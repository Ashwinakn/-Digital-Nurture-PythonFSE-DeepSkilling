-- First Normal Form (1NF)

-- The schema satisfies 1NF.
-- Every column contains atomic (single) values.
-- There are no repeating groups or multi-valued attributes.
-- Example violation:
-- phone_number = '9876543210,9876501234'

-- Second Normal Form (2NF)

-- The schema satisfies 2NF.
-- Every non-key attribute depends on the entire primary key.
-- In enrollments, student_id, course_id,
-- enrollment_date and grade depend on the enrollment record.

-- Third Normal Form (3NF)

-- The schema satisfies 3NF.
-- Department information is stored only in the departments table.
-- Students store only department_id.
-- Therefore no transitive dependency exists.

-- Example violation:
-- If dept_name were stored inside students table,
-- updating a department name would require changing
-- multiple student records.

--  3NF Analysis for Enrollments Table

Enrollments stores only:
enrollment_id
student_id
course_id
enrollment_date
grade

No attribute depends on another non-key attribute.
Therefore the table satisfies Third Normal Form (3NF).
